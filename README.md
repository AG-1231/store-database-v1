# store-database-v1
This SQL script establishes a relational database system for a game storefront by defining a structured schema across three distinct tables: Games, Customers, and Purchases. By utilizing SQLite syntax, the project demonstrates fundamental data architecture principles such as Data Normalization and the implementation of Primary and Foreign Keys to maintain strict relational integrity between inventory and user transactions. The included logic culminates in an advanced JOIN query, effectively showcasing the ability to bridge disparate data sets to produce detailed, human-readable reports on specific customer purchase histories.
